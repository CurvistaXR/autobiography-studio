# Public Research, Drafting, Cover, and Production

Read this file before web research, full drafting, cover generation, or final export.

## Public-source research

Obtain permission before searching. Use identifying attributes the user approves: location, organization, profession, school, approximate dates, or known publications.

For every candidate result, present:

- page title and publisher/domain
- URL and access date
- publication/event date when available
- identity signals that match the user
- identity conflicts or namesake risk
- exact proposed claim
- source type and confidence

Prefer first-party records and authoritative primary sources. For a consequential disputed claim, seek two independent reliable sources when available. A search snippet is a lead, not evidence. Do not adopt any result until user confirmation.

If identity remains ambiguous, mark `excluded` or `uncertain`; never merge a namesake's history into the book.

## Drafting rules

- Follow the approved outline and style card.
- Draft one chapter or bounded section at a time.
- Anchor scenes in confirmed time, place, people, and action.
- Attribute memory: “我记得”, “在我的印象里”, or equivalent when certainty is personal rather than documentary.
- Preserve conflicting accounts when they matter.
- Separate contemporaneous feelings from later interpretation.
- Do not create quotations from notes. Reconstructed dialogue requires user confirmation and transparent framing.
- Avoid legal conclusions, diagnoses, motives, or accusations not supported and approved.
- Maintain the user's voice while correcting repetition, chronology, and readability.

After each chapter, list unresolved facts and sensitive passages outside the prose. Update the ledgers before continuing.

## Cover routes

Create three concepts and obtain user confirmation.

### Authorized photo

Use only the specific image the user approves. Do not infer consent from the presence of a photo in a folder. Preserve identity and avoid misleading edits.

### Symbolic generated image

Base symbols on approved themes, places, objects, or eras. Do not depict unsupported uniforms, awards, institutions, wealth, hardship, age, family members, or historical settings. Do not use a real-person likeness unless supplied and authorized.

### Typographic fallback

Use `assets/typographic-cover.svg`. Replace title, subtitle, author, and optional years; select a restrained color palette that matches the edition. This route requires no image generation.

Check title readability at thumbnail size, safe margins, image rights, contrast, and consistency between cover metadata and title page.

## Book assembly

Recommended order:

1. Cover
2. Title page
3. Copyright and private-use/publication note
4. Dedication or epigraph when approved
5. Preface
6. Generated table of contents
7. Chapters
8. Photo section or inline photographs with captions
9. Afterword and acknowledgements
10. Life chronology
11. Optional public source note

Use real heading styles so the Word table of contents can update. Add page breaks deliberately. Compress images without destroying originals. Use accessible captions and alt text where supported.

## DOCX and PDF integrity

Use host-native document creation for `.docx` and PDF creation/rendering for `.pdf`. Do not rename Markdown, plain text, RTF, or HTML to these extensions.

Before delivery:

- reopen the DOCX and verify it is a valid ZIP-based Office document
- update or verify the table of contents
- render the PDF and inspect cover, contents, representative chapters, photo pages, and final pages
- check fonts, Chinese punctuation, widows/orphans, clipped text, blank pages, page numbers, headings, image resolution, and captions
- confirm file names and metadata match the approved title and edition

If a required capability is unavailable, deliver the manuscript source and explain the missing capability. Do not claim that Word or PDF exists.

## Final user confirmation

Show a compact list of title, subtitle, author name, edition, audience, chapter order, sensitive passages, public sources, photo permissions, cover, and export formats. Require user confirmation before final export.

For a public-ready delivery, copy only the approved book files, cover, and optional public source note to the share folder. Keep raw sources and working ledgers elsewhere.
